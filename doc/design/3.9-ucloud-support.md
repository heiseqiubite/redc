# UCloud 云厂商支持实现

## 概述

本文档记录在 redc 和 redc-gui 中添加 UCloud 云厂商支持的过程，供后续维护参考。

---

## 1. 后端配置 (mod/config.go)

### 1.1 添加 UCloud 配置结构体

文件: `mod/config.go`

```go
UCloud struct {
    PublicKey  string `yaml:"UCLOUD_PUBLIC_KEY" env:"UCLOUD_PUBLIC_KEY"`
    PrivateKey string `yaml:"UCLOUD_PRIVATE_KEY" env:"UCLOUD_PRIVATE_KEY"`
    Region     string `yaml:"region"`
    ProjectId  string `yaml:"project_id" env:"UCLOUD_PROJECT_ID"`
} `yaml:"ucloud"`
```

### 1.2 获取凭据函数

文件: `mod/config.go`

在 `GetProviderCredentials` 函数中添加:

```go
case "ucloud":
    return LoadedConfig.Providers.UCloud.PublicKey, LoadedConfig.Providers.UCloud.PrivateKey
```

## 2. Provider 配置 (mod/deployment_executor.go)

文件: `mod/deployment_executor.go`

```go
case "ucloud":
    return &ProviderConfig{
        Name:    "ucloud",
        Source:  "ucloud/ucloud",
        Version: "~> 0.0",
        Config: map[string]string{
            "region": region,
        },
    }, nil
```

## 3. 密码自动生成 (mod/case.go)

### 3.1 添加 UCloud 密码生成支持

文件: `mod/case.go`

在 `ensureProviderVars` 函数中添加:

```go
// 处理 UCloud
// UCloud 使用环境变量 UCLOUD_PUBLIC_KEY 和 UCLOUD_PRIVATE_KEY，不需要通过 -var 传递
if provider == "ucloud" {
    // 如果模板需要 instance_password 且用户未提供，则自动生成
    if vars["instance_password"] == "" {
        vars["instance_password"] = generateInstancePassword()
        gologger.Info().Msgf("已自动生成 UCloud 实例密码")
    }
}
```

## 5. GUI 凭据页面 (app.go)

### 5.1 获取凭据配置

文件: `app.go`

在 `GetProvidersConfig` 函数中添加 UCloud 凭据字段:

```go
{
    Name: "UCloud",
    Fields: map[string]string{
        "publicKey":  maskValue(conf.Providers.UCloud.PublicKey),
        "privateKey": maskValue(conf.Providers.UCloud.PrivateKey),
        "projectId":  conf.Providers.UCloud.ProjectId,
        "region":     conf.Providers.UCloud.Region,
    },
    HasSecrets: map[string]bool{
        "publicKey":  conf.Providers.UCloud.PublicKey != "",
        "privateKey": conf.Providers.UCloud.PrivateKey != "",
    },
},
```

### 5.2 保存凭据配置

文件: `app.go`

在 `SaveProvidersConfig` 函数中添加:

```go
case "UCloud":
    if v, ok := fields["publicKey"]; ok && v != "" {
        conf.Providers.UCloud.PublicKey = v
    }
    if v, ok := fields["privateKey"]; ok && v != "" {
        conf.Providers.UCloud.PrivateKey = v
    }
    if v, ok := fields["projectId"]; ok {
        conf.Providers.UCloud.ProjectId = v
    }
    if v, ok := fields["region"]; ok {
        conf.Providers.UCloud.Region = v
    }
```

## 9. 注意事项

1. **UCloud Provider 版本**: 建议使用固定版本如 `1.39.2`
2. **Project ID**: UCloud 需要填写项目 ID，不能使用 "Default"
3. **镜像**: 模板中使用 `data "ucloud_images"` 查询可用镜像
4. **密码**: redc 会自动生成符合 UCloud 要求的密码（8-30位，包含大小写字母、数字和特殊字符）
5. **凭据传递**: 通过环境变量 `UCLOUD_PUBLIC_KEY` 和 `UCLOUD_PRIVATE_KEY` 传递
