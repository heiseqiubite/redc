# 3.41 任务中心（Task Center）

## 概述

任务中心是从预定义场景（Cases）页面中拆分出来的独立运维页面，提供定时任务和周期性任务的统一管理能力。它位于侧边栏「运维工具」分组下，与控制台、资源管理、SSH 终端管理并列。

## 设计动机

- 预定义场景页面职责过重，同时承担场景管理和定时任务管理
- 定时任务是运维能力，应归入运维工具分组
- 用户需要周期性任务（每天/每周/自定义间隔）支持，原有的一次性定时任务无法满足

## 后端设计

### ScheduledTask 结构

```go
type ScheduledTask struct {
    ID             string    `json:"id"`
    CaseID         string    `json:"caseId"`
    CaseName       string    `json:"caseName"`
    Action         string    `json:"action"`       // "start" | "stop"
    ScheduledAt    time.Time `json:"scheduledAt"`
    Status         string    `json:"status"`        // "pending" | "completed" | "failed" | "cancelled"
    Error          string    `json:"error"`
    RepeatType     string    `json:"repeatType"`    // "once" | "daily" | "weekly" | "interval"
    RepeatInterval int       `json:"repeatInterval"` // 分钟数（仅 interval 模式使用）
    CompletedAt    *time.Time `json:"completedAt"`
}
```

### 数据库迁移

使用 `ALTER TABLE ADD COLUMN` 方式添加新字段，SQLite 在列已存在时静默失败，保证向后兼容：

```go
func (ts *TaskScheduler) InitDB() {
    // ... 原有建表逻辑 ...
    
    // 迁移：添加周期任务字段
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN repeat_type TEXT DEFAULT 'once'")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN repeat_interval INTEGER DEFAULT 0")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN completed_at DATETIME")
}
```

### 周期任务自动续期

任务执行完成后（无论成功或失败），`executeTask()` 调用 `scheduleNextRepeat()`：

```go
func (ts *TaskScheduler) scheduleNextRepeat(task *ScheduledTask) {
    if task.RepeatType == "once" || task.RepeatType == "" {
        return
    }
    
    var nextTime time.Time
    switch task.RepeatType {
    case "daily":
        nextTime = task.ScheduledAt.Add(24 * time.Hour)
    case "weekly":
        nextTime = task.ScheduledAt.Add(7 * 24 * time.Hour)
    case "interval":
        nextTime = task.ScheduledAt.Add(time.Duration(task.RepeatInterval) * time.Minute)
    }
    
    // 快进到未来时间（如果计算出的下一次时间已过去）
    for nextTime.Before(time.Now()) {
        switch task.RepeatType {
        case "daily":
            nextTime = nextTime.Add(24 * time.Hour)
        case "weekly":
            nextTime = nextTime.Add(7 * 24 * time.Hour)
        case "interval":
            nextTime = nextTime.Add(time.Duration(task.RepeatInterval) * time.Minute)
        }
    }
    
    // 创建新任务（新 ID，继承 repeat 配置）
    ts.AddTaskWithRepeat(task.CaseID, task.CaseName, task.Action, nextTime,
        task.RepeatType, task.RepeatInterval)
}
```

### 核心 API

| 函数 | 签名 | 描述 |
|------|------|------|
| `AddTaskWithRepeat` | `(caseID, caseName, action string, scheduledAt time.Time, repeatType string, repeatInterval int)` | 创建任务（支持周期配置） |
| `ListAllTasksFromDB` | `() []ScheduledTask` | 从 DB 读取所有任务（含最近 7 天历史），按 scheduled_at 降序 |
| `ScheduleTaskWithRepeat` | App 层封装 | Wails 绑定，调用 `AddTaskWithRepeat` |
| `ListAllScheduledTasks` | App 层封装 | Wails 绑定，调用 `ListAllTasksFromDB` |

### ListAllTasksFromDB 与内存 map 的区别

- `ts.tasks` (内存 map)：仅保存 `pending` 状态的任务，用于调度循环
- `ListAllTasksFromDB()`：直接读 SQLite，包含 completed/failed/cancelled 历史记录（最近 7 天）

## 与 Cases 页面的关系

- Cases 页面保留 `ScheduleDialog` 组件（单个场景的快捷定时按钮）
- Cases 页面移除了 `ScheduledTasksManager` 嵌入组件
- ScheduleDialog 创建的任务仍然通过 `ScheduleTask()` API（代理到 `ScheduleTaskWithRepeat`，默认 once）
- 所有任务统一在任务中心页面查看和管理

## 注意事项

1. **SQLite 并发**：`TaskScheduler` 使用 `sync.Mutex` 保护所有读写操作
2. **时间快进**：周期任务如果程序关闭期间错过了多次执行，续期时会快进到下一个未来时间点，不会补执行
3. **ListCases 返回格式**：场景的 ID/Name 字段存在大小写不一致（`id`/`Id`, `name`/`Name`），TaskCenter 前端通过 `c.id || c.Id` 兼容处理
4. **Svelte 5 限制**：不能在任意位置使用 `{@const}`，需要在 `{#each}` / `{#if}` 块内部或改为内联表达式
5. **DB 迁移安全性**：`ALTER TABLE ADD COLUMN` 在 SQLite 中如果列已存在会返回错误，但 `db.Exec` 忽略该错误，保证重复启动安全
