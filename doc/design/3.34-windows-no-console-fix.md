# Windows 黑框问题修复设计文档

## 问题描述

redc-gui 在 Windows 下以 GUI 模式运行时（`wails build` 生成的可执行文件，链接参数 `-H windowsgui`），每次执行 terraform 命令（init、plan、apply、destroy）都会短暂弹出黑色 cmd.exe 控制台窗口，严重影响用户体验。

开发模式（`wails dev`）不出现此问题，因为 dev 进程本身在终端中运行，子进程可继承父进程的控制台。

上游 issue：https://github.com/hashicorp/terraform-exec/issues/570

## 根本原因分析

### Windows 进程控制台分配机制

Windows 在创建子进程时遵循以下规则：
- 父进程有控制台 → 子进程继承控制台（不弹新窗口）
- 父进程无控制台（GUI 应用） → 子进程被 Windows 自动分配一个新的**可见**控制台窗口

redc-gui 用 `-H windowsgui` 编译，运行时没有控制台，因此每个 terraform 子进程都会触发新控制台弹出。

### 为何 TF_IN_AUTOMATION、TF_INPUT 等环境变量无效

这些变量只控制 Terraform **内部行为**（是否等待用户输入、是否显示进度条），不影响 Windows 系统级的控制台窗口分配。

### 唯一正确解法

在调用 `exec.Command` 后、`cmd.Start()` 前，设置：
```go
cmd.SysProcAttr = &syscall.SysProcAttr{
    HideWindow: true,
}
```

`HideWindow: true` 对应 Windows API 的 `CREATE_NO_WINDOW`（`0x08000000`）标志，告诉内核创建进程时不分配可见控制台。stdout/stderr 通过管道（pipe）捕获，功能完全不受影响。

## terraform-exec 未处理的原因

`terraform-exec` 的 `cmd_default.go`（构建标签 `!linux`，涵盖 Windows 和 macOS）中没有设置任何 `SysProcAttr`。Linux 版本（`cmd_linux.go`）设置了 `Pdeathsig` / `Setpgid`，但没有 Windows 专属版本。

## 解决方案：本地 patch

由于上游长期无响应，采用 Go modules 的 `replace` 机制本地 patch：

```
patches/terraform-exec/          ← 从 go module cache 复制并修改
└── tfexec/
    ├── cmd_default.go            ← 构建标签: !linux → !linux && !windows
    └── cmd_windows.go            ← 新增 Windows 专属实现
```

`go.mod` 末尾：
```
replace github.com/hashicorp/terraform-exec => ./patches/terraform-exec
```

### cmd_windows.go 核心改动

```go
//go:build windows

func (tf *Terraform) runTerraformCmd(ctx context.Context, cmd *exec.Cmd) error {
    // HideWindow 阻止 Windows 为无控制台 GUI 进程的子进程分配新窗口
    cmd.SysProcAttr = &syscall.SysProcAttr{
        HideWindow: true,
    }
    // ... 其余逻辑与 cmd_default.go 完全相同
}
```

### 为何选择 replace 而非 wrapper 代理

用户朋友提出的"redc 调 redc CLI 代理 terraform"思路也可行，但：

| 方案 | 复杂度 | 维护成本 | 跨场景覆盖 |
|------|--------|----------|------------|
| `replace` patch | 极低（2个文件） | 极低 | 覆盖所有 tf 调用 |
| 代理程序 | 高（需实现 IPC/CLI 接口） | 中 | 需改所有调用点 |

patch 方案只改 2 个文件，其余代码零改动。

## 移除条件

等 terraform-exec 官方合并修复后：
1. 删除 `go.mod` 中的 `replace` 行
2. 删除 `patches/terraform-exec/` 目录
3. `go mod tidy`

## 文件变更

| 文件 | 变更 |
