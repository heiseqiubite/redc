# 3.38 模板测试沙箱

## 功能概述

在本地模板页面提供「语法检查」功能，通过 `terraform init` + `terraform validate` 对模板进行 dry-run 验证，检查语法错误和参数合法性，无需实际连接云厂商。

## 背景

用户在本地编辑或新建模板后，无法提前验证模板是否存在语法错误、变量引用错误、类型不匹配等问题，只有在实际启动场景（terraform plan/apply）时才会发现错误。测试沙箱功能让用户在部署前快速发现问题。

## 实现方案

### 后端

#### TfValidate（mod/tf.go）

新增 `TfValidate(Path string) (*tfjson.ValidateOutput, error)`：
1. 创建 `TerraformExecutor`
2. 调用 `te.Validate(ctx)` — 底层调用 `terraform validate -no-color -json`
3. 返回结构化的 `tfjson.ValidateOutput`

`TerraformExecutor` 新增 `Validate` 方法（mod/terraform.go）：
```go
func (te *TerraformExecutor) Validate(ctx context.Context) (*tfjson.ValidateOutput, error) {
    return te.tf.Validate(ctx)
}
```

#### ValidateTemplate（app_templates_registry.go）

`ValidateTemplate(templateName string) (*TemplateValidateResult, error)`：
1. 通过 `GetTemplatePath` 获取模板路径
2. **Step 1**: 执行 `terraform init` — 下载 provider（validate 必须先 init）
3. **Step 2**: 执行 `terraform validate` — 检查语法和参数
4. 将 `tfjson.ValidateOutput` 转换为前端友好的 `TemplateValidateResult`

如果 init 失败，直接返回带有 init 错误的结果（不中断为异常）。

#### 返回数据结构

```go
type TemplateValidateResult struct {
    Valid        bool                         `json:"valid"`
    ErrorCount   int                          `json:"error_count"`
    WarningCount int                          `json:"warning_count"`
    Diagnostics  []TemplateValidateDiagnostic `json:"diagnostics"`
}

type TemplateValidateDiagnostic struct {
    Severity string `json:"severity"`  // "error" | "warning"
    Summary  string `json:"summary"`
    Detail   string `json:"detail"`
    Filename string `json:"filename"`
    Line     int    `json:"line"`
}
```

### 前端

#### 触发方式

两个入口：
1. **模板列表** — 每个模板操作区新增「语法检查」按钮（emerald 绿色）
2. **模板编辑器** — 编辑器顶部栏新增「语法检查」按钮（amber 橙色），方便编辑后立即验证

#### 验证结果弹窗

- **加载中**：灰色圆形 spinner + "验证中..." 文字
- **验证通过**：绿色 ✓ 图标 + "验证通过" + 成功提示
- **验证失败**：红色 ⚠ 图标 + "验证失败" + 错误/警告计数
  - 每条诊断信息以卡片展示：
    - ERROR 红色标签 / WARN 橙色标签
    - summary（主要错误描述）
    - detail（详细信息，可折行）
    - filename:line（源文件位置）
  - 诊断列表最大高度 300px，支持滚动

### terraform validate 说明

`terraform validate` 是 Terraform 内置的语法检查命令：
- 检查 HCL 语法正确性
- 检查变量引用是否存在
- 检查类型约束是否满足
- 检查 provider/resource 块结构
- **不需要** 云厂商凭证（不连接 API）
- **需要** 先执行 `terraform init` 下载 provider schema

这使得它非常适合作为"沙箱"验证工具 — 用户无需配置凭证即可验证模板语法。
