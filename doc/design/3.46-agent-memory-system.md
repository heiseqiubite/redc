# 3.46 Agent 记忆系统

## 概述

Agent 记忆系统让 AI Agent 能够记住跨对话的操作经验和用户偏好。每次对话结束后，系统自动提取值得记住的经验教训，存入 SQLite 数据库；下次对话开始时，自动注入历史经验到系统提示词中。

## 问题背景

AI Agent 每次对话独立，不知道用户历史操作偏好。典型问题：
- 用户多次部署都用阿里云，但 Agent 每次都重新问
- AWS 模板选错架构（ARM vs x86），同样的错误反复出现
- 用户偏好特定的模板/配置，Agent 无法感知

## 架构设计

### 数据流

```
对话结束 → extractMemories() → LLM 提取经验 → agent_memory 表
对话开始 → GetMemoryContext() + GetUsageStats() → 拼接到系统提示词
```

### 双源记忆

1. **经验提取（LLM）**：对话结束后异步调用 LLM，从对话内容中提取通用经验教训和用户偏好
2. **使用统计（实时）**：扫描已有 cases 数据，统计模板使用频率和云厂商分布

两者合并注入系统提示词的 `## 用户偏好与历史经验` 章节。

### 存储

- SQLite 数据库 `memory.db`，存储在项目目录下
- 表结构：`agent_memory(id, project, category, content, source, created_at)`
- category：`lesson`（经验教训）、`preference`（使用偏好）、`note`（备注）
- 上限 50 条，超出自动淘汰最老记录
- 写入前去重（相同 project + content 不重复写入）

### 记忆提取

- 在 `runAgentLoop` 两个退出点（正常完成 / 达到最大轮次）触发
- 异步 goroutine 执行，不阻塞主流程
- 构建对话摘要：只取 user + assistant 消息，每条截断 500 字符，总计 4000 字符
- LLM 提取 prompt 要求最多 3 条，格式 `category|content`
- 提取失败静默忽略

### 配置开关

- `AIConfig.EnableMemory *bool`，nil 时默认开启
- 凭据管理页 AI 配置区提供 emerald 风格 toggle
- 关闭后：不注入记忆上下文，不提取新记忆

## 前端页面

AgentMemory.svelte 提供：
- 统计卡片（总数、经验教训数、使用偏好数）
- 记忆列表：分类标签（颜色区分）、内容、时间
- 单条删除（hover 显示）、全部清空（带确认弹窗）
- 空状态：🧠 图标 + 提示文案

## 注意事项

1. `GetMemoryContext()` 不包含顶级标题，由调用方 `app_ai_chat.go` 统一添加 `## 用户偏好与历史经验` 标题
2. 提取使用 `ChatWithTools(ctx, messages, nil)` 调用 LLM，因为项目中没有独立的 completion API
3. 记忆提取是异步的，不会影响用户体验
4. 统计数据实时计算不存储，确保与实际 cases 数据一致
5. 记忆注入覆盖所有对话模式：`AIChatStream`（自由对话/生成模板/推荐场景/成本优化/报错分析）和 `runAgentLoop`（Agent 助手/部署助手），共 7 个模式。`AIChatStream` 中通过读取 `a.project.ProjectName` 获取当前项目名，注入逻辑与 `runAgentLoop` 一致
