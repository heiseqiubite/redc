# 版本更新检测功能实现

## 概述

本文档记录 redc 桌面应用中版本更新检测功能的实现，供后续维护参考。

---

## 1. 功能设计

### 1.1 触发方式

用户点击界面左下角的版本号，触发版本检测并跳转到关于页面。

### 1.2 检测流程

1. 点击版本号 → 触发 `CheckForUpdates` 后端 API
2. 后端调用 GitHub API 获取最新版本
3. 返回检测结果到前端
4. 关于页面显示检测状态和结果

### 1.3 GitHub API

使用 GitHub REST API 获取最新 release：

```
GET https://api.github.com/repos/wgpsec/redc/releases/latest
```

响应中的 `tag_name` 字段即为最新版本号。

## 2. 后端实现 (app.go)

### 2.1 数据结构

```go
type VersionCheckResult struct {
    CurrentVersion string `json:"currentVersion"`
    LatestVersion  string `json:"latestVersion"`
    HasUpdate     bool   `json:"hasUpdate"`
    DownloadURL    string `json:"downloadURL"`
    Error         string `json:"error"`
}
```

### 2.2 CheckForUpdates 函数

```go
func (a *App) CheckForUpdates() (VersionCheckResult, error)
```

实现逻辑：

1. 调用 GitHub API `GET https://api.github.com/repos/wgpsec/redc/releases/latest`
2. 解析响应获取 `tag_name`（最新版本号）
3. 使用 `compareVersions()` 比较当前版本和最新版本
4. 返回检测结果

### 2.3 版本比较函数

```go
func compareVersions(current, latest string) int
```

实现逻辑：
- 分割版本号为数组（如 "3.0.2" → ["3", "0", "2"]）
- 逐位比较数字大小
- 返回 -1（当前较旧）、0（相同）、1（当前较新）

## 3. 前端实现

### 3.1 状态管理 (App.svelte)

```javascript
let updateStatus = $state({ checking: false, result: null });

async function checkForUpdates() {
    updateStatus.checking = true;
    updateStatus.result = null;
    try {
        const result = await CheckForUpdates();
        updateStatus.result = result;
    } finally {
        updateStatus.checking = false;
    }
}
```

### 3.2 Sidebar 触发 (Sidebar.svelte)

```svelte
<button
    onclick={() => { 
        if (onCheckUpdate) onCheckUpdate(); 
        onTabChange('about'); 
    }}
>
    {version} by WgpSec
</button>
```

点击版本号时：
1. 调用 `onCheckUpdate` 触发版本检测
2. 跳转到关于页面（`onTabChange('about')`）

### 3.3 关于页面展示 (About.svelte)

```svelte
<!-- 检测中 -->
{#if updateStatus && updateStatus.checking}
    <svg class="animate-spin">...</svg>
{:else if updateStatus && updateStatus.result}
    <!-- 有新版本 -->
    {#if updateStatus.result.hasUpdate}
        <button onclick={() => BrowserOpenURL(result.downloadURL)}>
            {result.latestVersion} 可更新
        </button>
    {:else}
        <span>已是最新版本</span>
    {/if}
{:else}
    <!-- 默认状态 -->
    <button onclick={() => onCheckUpdate()}>检查更新</button>
{/if}
```

三种状态：

| 状态 | 显示 |
|------|------|
| 检测中 | 旋转加载图标 |
| 有新版本 | 版本号 + "可更新" 按钮，点击跳转 GitHub |
| 已是最新 | "已是最新版本" |
| 默认 | "检查更新" 按钮 |

## 5. 注意事项

### 5.1 版本号格式

- 当前版本从 `mod/flag.go` 的 `Version` 变量读取
- GitHub 返回格式：`v3.0.2`（带 v 前缀）
- 比较时去除前缀：`v3.0.2` → `3.0.2`

### 5.2 网络问题

- 如果无法连接 GitHub，返回错误信息 "无法连接 GitHub"
- 前端显示错误状态

### 5.3 下载链接

- 下载地址固定为：`https://github.com/wgpsec/redc/releases`
- 可根据需要修改为具体的 release 页面

### 5.4 版本号定义位置

当前版本号定义在 `mod/flag.go`：

```go
var (
    // ...
    Version = "v3.0.8"  // 以实际代码为准
)
```
