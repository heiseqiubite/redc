# 3.37 本地模板编辑器

## 功能概述

允许用户直接在 redc-gui 中新建 Terraform 模板目录，从空白或预设脚手架开始编写，无需外部工具或推送到 GitHub 仓库。同时增强现有模板编辑器，支持新增和删除文件。

## 背景

此前用户只能通过以下方式获取模板：
1. 从仓库 pull 下载
2. 导入 zip 包
3. 克隆已有模板

无法在 GUI 内从零开始创建模板。

## 实现方案

### 后端

#### CreateLocalTemplate(name string, scaffold string) error

位置：`app_templates_registry.go`

流程：
1. 校验模板名称：正则 `^[a-zA-Z0-9][a-zA-Z0-9_\-/]*[a-zA-Z0-9]$`，拒绝 `..` 路径穿越
2. 检查 `TemplateDir/name/` 是否已存在（已存在则报错）
3. 创建目录 `os.MkdirAll`
4. 根据 scaffold 参数写入脚手架文件
5. 触发 `emitRefresh()` 通知前端刷新

脚手架模式：
- `blank`：`main.tf`（注释指引）+ `variables.tf` + `outputs.tf` + `case.json`
- `with-userdata`：同 blank + `userdata` 初始化脚本

所有 `main.tf` 内容为注释示例，不绑定任何云厂商，用户自由选择 provider。

#### DeleteTemplateFile(templateName string, fileName string) error

位置：`app_templates_registry.go`

流程：
1. 通过 `GetTemplatePath` 获取模板路径
2. 校验 fileName 不含 `..`、`/`、`\`（防路径穿越）
3. `os.Remove` 物理删除文件

此方法补充了 `SaveTemplateFiles` 的能力（后者只写入不删除）。

### 前端

#### 新建模板对话框

位置：`LocalTemplates.svelte`

- 工具栏新增绿色「新建模板」按钮
- 弹出对话框包含：
  - 模板名称输入（前端校验格式）
  - 脚手架选择卡片（空白 / 含 Userdata）
  - 错误提示区
- 创建成功后自动打开模板编辑器

#### 编辑器文件管理增强

- 侧边栏文件列表每个文件右侧增加「×」删除按钮（hover 显示）
- 删除前弹出确认对话框（z-index: 60，高于编辑器的 z-50）
- 确认后调用 `DeleteTemplateFile` 物理删除
- 侧边栏底部增加「+ 新建文件」按钮
- 点击后显示 inline 输入框，输入文件名，回车确认或 ESC 取消
- 新文件以空内容添加到 files map，保存时由 `SaveTemplateFiles` 写入磁盘

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 脚手架内容 | 纯注释指引 | 用户选择自己的云厂商，不强制绑定 |
| 文件删除时机 | 点击确认后立即物理删除 | 避免保存时混淆；编辑器中保存仅写入现有文件 |
| 模板名校验 | `[a-zA-Z0-9_\-/]` | 允许斜杠分组（如 `myteam/ecs`），防路径穿越 |
| 脚手架种类 | 2 种（blank + with-userdata） | 简洁实用，不过度复杂 |
| 新文件添加 | 编辑器 sidebar inline 输入 | 无需额外对话框，交互流畅 |
| 文件删除确认 | 独立确认对话框 | 防误触，用户明确确认后才删除 |
