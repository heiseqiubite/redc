# 3.39 GUI UX 优化方案

## 概述

v3.1.1 对 GUI 进行了系统性 UX 优化，解决了导航效率低、操作无反馈、信息密度大等问题。本文档记录各优化项的设计逻辑和实现方式，供后续维护参考。

---

## 1. 侧边栏分组导航

### 问题
14 个平铺菜单项导致导航区域过长，用户找不到目标功能。

### 方案
将菜单项按功能归类为 5 个可折叠分组 + 1 个独立项：

| 分组 | 包含菜单项 |
|------|-----------|
| （独立）仪表盘 | Dashboard |
| 部署管理 | 预定义场景、自定义部署、编排管理 |
| 运维工具 | SSH 终端、终端日志、云资源、专项模块 |
| 模板管理 | 模板仓库、本地模板 |
| AI 助手 | AI 对话、AI 集成 |
| 系统 | 设置、凭据管理、关于 |

### 实现细节

**文件**: `frontend/src/components/Sidebar/Sidebar.svelte`

**数据结构**: `navSections` 数组，每项为 `{ type: 'item' }` 或 `{ type: 'group', key, labelKey, items }`.

**折叠持久化**: 使用 `localStorage` key `redc-sidebar-collapsed`，值为 JSON 对象如 `{ deploy: false, ops: true }`.

**自动展开**: 通过 `$effect` 检测当前 `activeTab` 所属分组是否折叠，若折叠则自动展开，确保活跃项始终可见。

**i18n**: 分组标题使用 `navGroupDeploy`、`navGroupOps`、`navGroupTemplate`、`navGroupAI`、`navGroupSystem` 键。

## 2. 场景列表搜索与分页

### 问题
Cases.svelte 2200+ 行，场景多时全量渲染卡顿，无搜索功能。

### 方案
三层过滤链 + 分页：

```
cases (原始数据)
  → filteredCases (状态筛选 + 标签筛选 + 搜索关键词)
    → paginatedCases (当前页切片)
```

### 实现细节

**文件**: `frontend/src/components/Cases/Cases.svelte`

**搜索**: 顶部搜索栏，匹配 `name`、`id`、`type`、`tags` 字段，大小写不敏感。

**状态快筛**: 4 个 tab 标签（全部/运行中/已停止/错误），每个显示对应数量角标。状态值对应 `mod/entity.go` 中定义的 `running`、`stopped`、`error`。

**分页**: 每页 20 条。分页控件包含首页/上一页/页码指示器/下一页/末页。`paginatedCases` 通过 `$derived` 从 `filteredCases` 切片。

**页码重置**: 使用 `_prevFilterKey` 追踪模式——将搜索词+状态+标签拼接为 key，key 变化时重置到第 1 页，避免 onMount 时的误重置。

**全选行为**: `toggleSelectAll` 只影响当前页的 case ID，不影响其他页已选中的项。

## 3. 全局 Toast 通知系统

### 问题
各页面使用浏览器原生 `alert()` 弹窗，阻塞操作流程，体验差。通知样式不统一。

### 方案
抽取全局 Toast 组件，统一 4 种通知样式。

### 实现细节

**Store**: `frontend/src/lib/toast.js`

```javascript
// 发布-订阅模式
toast.success(msg, duration?)  // 绿色，默认 3500ms
toast.error(msg, duration?)    // 红色，默认 5000ms
toast.warning(msg, duration?)  // 黄色，默认 3500ms
toast.info(msg, duration?)     // 蓝色，默认 3500ms
```

- 内部维护 `_toasts` 数组 + `_listeners` 数组
- 每条 toast 有递增 ID，支持 `removeToast(id)` 手动关闭
- `onToastChange(fn)` 返回 unsubscribe 函数

**组件**: `frontend/src/components/Toast/Toast.svelte`

- 固定定位右上角 (`fixed top-4 right-4 z-[9999]`)
- slide-in 动画 (`@keyframes slideIn`)
- 4 种样式使用不同背景色和图标
- `onMount` 订阅 store，`onDestroy` 取消订阅

**挂载**: 在 `App.svelte` 顶层引入 `<Toast />`，全局生效。

**替换范围**: 所有 `.svelte` 文件中的 `alert()` 调用（共 17 处），分布在 Sidebar、Cases、CustomDeployment、CustomDeploymentList。

## 4. 仪表盘快捷操作

### 问题
仪表盘只展示数据统计，无操作入口，用户需切到其他页面才能执行操作。

### 方案
三项增强：

1. **最近场景快捷按钮**: hover 时显示操作按钮（运行中→停止/SSH，已停止→启动）
2. **快速创建入口**: 标题栏蓝色"快速创建"按钮 + 空状态引导按钮
3. **网络诊断跳转**: 有 FAIL 项时底部显示"前往凭据管理 →"链接

### 实现细节

**文件**: `frontend/src/components/Dashboard/Dashboard.svelte`

**快捷按钮**: 使用 CSS `group` + `hidden group-hover:flex` 实现 hover 显示。按钮调用 `StartCase(id)` / `StopCase(id)` 并通过 `actionLoading` 对象追踪每个 case 的加载状态。操作结果通过 toast 通知。

**创建入口**: 跳转到 `customDeployment` tab（注意 tab 名是 camelCase 而非 kebab-case）。

**网络诊断**: `{#if networkChecks.some(item => !item.ok)}` 条件渲染跳转链接。

## 5. 编排管理页面重构

### 问题
原编排管理页面布局杂乱，操作按钮层级不清晰，执行结果只在终端显示，用户不知道状态。

### 方案
双栏布局 + 状态系统 + 日志面板。

### 实现细节

**文件**: `frontend/src/components/Compose/Compose.svelte`

**布局**: 左栏（配置区）+ 右栏（服务列表 + 操作按钮 + 日志）。

**配置来源**: radio 按钮切换 `template`（从模板仓库选择）或 `file`（手动指定文件路径），存储在 `configSource` state。

**服务状态徽章**: `getStatusBadge(status)` 返回对应颜色样式。状态通过后端 `ComposePreview` 的 `Status` 字段获取（`app.go` 中 `ComposeServiceSummary` 结构体新增 `Status string` 字段）。

**按钮层级**:
- 主按钮（绿色）: 部署 (compose up)
- 危险按钮（红色）: 销毁 (compose down)，需二次确认弹窗
- 辅助按钮: 查看拓扑

**状态持久化**: `frontend/src/lib/composeState.js` 保存执行状态，切出页面再回来仍可见。

**日志面板**: 通过 Wails `EventsOn('compose-log', ...)` 接收实时日志。

## 6. 操作反馈动画统一

### 问题
部分按钮点击后无状态变化，用户不确定操作是否生效。

### 方案
统一模式: `按钮原文` → `🔄 spinner + "保存中..."` → `✓ "已保存"` → 恢复原样（1.5s）。

### 实现细节

**可复用组件**: `frontend/src/components/UI/AsyncButton.svelte`

```svelte
<!-- 内部管理 phase: 'idle' | 'loading' | 'success' -->
<!-- onclick 返回 Promise，自动切换状态 -->
<AsyncButton onclick={handleSave} class="...">保存</AsyncButton>
```

**内联模式**: 对已有 `xxxSaving` 状态的按钮，新增 `xxxSaved` 状态变量：

```javascript
// 在 try 块成功后：
proxySaved = true;
setTimeout(() => { proxySaved = false; }, 1500);
```

按钮模板中使用三段式渲染：

```svelte
{#if proxySaving}
  <svg class="animate-spin ...">...</svg> {t.saving}
{:else if proxySaved}
  <svg class="...">✓</svg> {t.saved}
{:else}
  {t.saveProxy}
{/if}
```

**覆盖范围**:

| 页面 | 按钮 | 方式 |
|------|------|------|
| Settings | 代理保存、Terraform 镜像保存 | 内联 spinner + ✓ saved |
| Credentials | profile 创建/保存/删除 | 内联 spinner + ✓ |
| Credentials | 凭据保存、AI 配置保存 | 内联 spinner |
| Resources | 资源刷新、余额查询 | 内联 spinner |
| Registry | 模板下载 | 已有 pulling spinner ✓ |

## 8. 设计原则

1. **最小侵入**: 优先在现有组件上添加状态变量，而非重写组件
2. **一致性**: 所有 spinner 使用同一 SVG（animate-spin），所有 ✓ 使用同一 checkmark SVG
3. **颜色规范**: 遵循 `doc/design/3.8-gui-design.md` — Primary=blue-600, Success=emerald-500, Danger=red-500
4. **持久化策略**: 导航折叠→localStorage，执行状态→内存 store（composeState.js），搜索/分页→组件内 state
5. **Svelte 5 模式**: 使用 `$state`、`$derived`、`$derived.by()`、`$effect`，对象需重新赋值触发响应式
