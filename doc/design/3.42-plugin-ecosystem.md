# 3.42 插件生态系统 (Plugin Ecosystem)

## 概述

RedC 插件系统允许用户通过独立的插件包扩展 RedC 的功能，包括 Terraform 模板、Userdata 脚本、生命周期钩子脚本等。插件通过插件市场（redc-template 仓库）、Git 仓库 URL 或本地路径安装，支持启用/禁用、配置、版本更新、卸载。

原有的内置模块（`mod/modules.go` 中的 `gen_clash_config`、`upload_r2`、`chang_dns`）已完全迁移为外部插件，通过 `redc_plugins` 字段与场景绑定。

## 核心架构

### 插件目录结构

```
~/.redc/plugins/
├── redc-plugin-clash-config/
│   ├── plugin.json            # 插件清单（必须）
│   ├── hooks/                 # 钩子脚本（可选）
│   │   └── post-apply.sh
│   ├── config.json            # 用户配置（自动生成）
│   └── .disabled              # 存在时表示已禁用
├── redc-plugin-upload-r2/
│   └── ...
└── redc-plugin-cloudflare-dns/
    └── ...
```

### plugin.json 清单

```json
{
  "name": "redc-plugin-clash-config",
  "version": "1.0.1",
  "description": "场景部署后自动生成 Clash 代理配置文件",
  "description_en": "Auto-generate Clash proxy config after deployment",
  "author": "wgpsec",
  "homepage": "https://github.com/wgpsec/redc-template",
  "category": "proxy",
  "tags": ["clash", "proxy", "shadowsocks"],
  "min_redc_version": "3.1.0",
  "capabilities": {
    "hooks": {
      "post-apply": "hooks/post-apply.sh"
    }
  },
  "config_schema": {
    "port": { "type": "string", "required": false, "description": "SS 端口", "default": "" },
    "password": { "type": "string", "required": false, "description": "SS 密码", "default": "" }
  }
}
```

### 场景-插件绑定 (`redc_plugins`)

场景模板的 `case.json` 通过 `redc_plugins` 字段声明依赖哪些插件：

```json
{
  "name": "proxy",
  "version": "1.1.6",
  "description": "AWS 多节点代理场景",
  "redc_plugins": "redc-plugin-clash-config,redc-plugin-upload-r2",
  "template": "preset"
}
```

- 场景创建时从模板 `case.json` 读取 `redc_plugins`，存入 `Case.Plugins` 字段（protobuf field 13）
- 钩子执行时仅运行 `AllowedPlugins` 列表中的插件，空列表 = 不运行任何插件（显式 opt-in）
- **执行顺序按 `redc_plugins` 声明顺序**（如先 clash-config 生成文件，再 upload-r2 上传）

### 插件输出协议 (`REDC_OUTPUT`)

钩子脚本通过 stdout 输出结构化数据，供 GUI 展示：

```bash
echo "REDC_OUTPUT:clash_config_file=/path/to/config.yaml"
echo "REDC_OUTPUT:clash_node_count=10"
```

- `executeHook` 解析所有 `REDC_OUTPUT:key=value` 行
- `RunHooks` 累积所有插件输出，写入 `{casePath}/plugin_outputs.json`
- `GetCaseOutputs` 读取并合并到 terraform outputs 中，在 GUI 输出信息区统一展示

## 后端实现

### 包结构 (`mod/plugin/`)

| 文件 | 职责 |
|------|------|
| `plugin.go` | 核心结构体：PluginManifest、Plugin、PluginManager、ConfigField |
| `loader.go` | LoadAll() 扫描插件目录、GetTemplatePaths/GetUserdataPaths/GetHooks |
| `installer.go` | Install（git clone / ZIP / 本地复制）、Uninstall、Enable/Disable、Update、ReinstallFromURL、SaveConfig |
| `hooks.go` | HookEntry、HookContext、RunHooks（按声明顺序执行 + 容错）、executeHook（bash 执行 + 环境变量注入 + REDC_OUTPUT 解析） |
| `registry.go` | FetchRegistry 从远程获取插件索引（默认 `redc.wgpsec.org`） |

### PluginManager 关键方法

```go
NewPluginManager(pluginsDir string) *PluginManager  // 空串则使用 ~/.redc/plugins/
LoadAll() error                                      // 扫描并加载所有插件
List() []*Plugin                                     // 返回已安装插件列表
Get(name string) (*Plugin, bool)                     // 按名查找
Install(source string) (string, error)               // 从 git URL、ZIP URL 或本地路径安装
Uninstall(name string) error                         // 删除插件目录
Enable(name string) error                            // 删除 .disabled 标记
Disable(name string) error                           // 创建 .disabled 标记
Update(name string) (string, error)                  // git pull 或从 registry 重新下载
ReinstallFromURL(name, url string) (string, error)   // 删除旧版 + 重新安装（保留配置和启用状态）
SaveConfig(name string, config map[string]interface{}) error
GetHooks(hookPoint string) []HookEntry               // 获取某钩子点的所有脚本
RunHooks(hookPoint string, ctx *HookContext) error    // 按声明顺序执行，单个失败不阻断其余
LoadPluginOutputs(casePath string) map[string]string  // 读取 plugin_outputs.json
```

### 启用/禁用机制

使用文件标记 `.disabled`：
- 存在 `<pluginDir>/.disabled` → 插件已禁用
- `Disable()` 创建该文件，`Enable()` 删除该文件
- 不依赖数据库，插件目录移动后仍保持状态

### 钩子执行

6 个钩子点：`pre-plan`、`post-plan`、`pre-apply`、`post-apply`、`pre-destroy`、`post-destroy`

钩子脚本通过 `bash <script>` 执行，5 分钟超时，注入以下环境变量：

```bash
REDC_HOOK_POINT=post-apply
REDC_PLUGIN_NAME=redc-plugin-clash-config
REDC_PLUGIN_DIR=/path/to/plugin
REDC_CASE_NAME=my-case
REDC_CASE_PATH=/path/to/case
REDC_CASE_TEMPLATE=aws/proxy
REDC_CASE_STATE=running
REDC_OUTPUT_JSON='{"ecs_ip":{"value":["1.2.3.4"]}}'
REDC_PLUGIN_CONFIG='{"port":"60001"}'
REDC_PLUGIN_CONFIG_PORT=60001
```

**容错机制：** 单个插件钩子执行失败时记录 warning 日志并跳过，继续执行其余插件，已成功插件的输出仍会保存。

### 与 Case 生命周期的集成

在 `mod/entity.go` 的 Case 结构体中：
- `pluginHookRunner` 回调字段：由 App 层通过 `setupPluginHooks()` 注入
- `Plugins string` 字段：存储场景声明的插件列表（逗号分隔），持久化到 protobuf

App 层在 `StartCase`、`StopCase`、`CreateAndRunCase` 时均调用 `setupPluginHooks()` 注入回调。

钩子调用位置（`mod/case.go`）：
- `TfApply()`: pre-apply → TfApply → post-apply
- `TfDestroy()`: pre-destroy → TfDestroy → post-destroy

### 更新机制

`Update()` 方法支持两种更新路径：
1. **Git 插件**（有 `.git` 目录）：执行 `git pull --ff-only`
2. **ZIP 安装的插件**（无 `.git`）：自动查询远程 registry，找到对应 URL，调用 `ReinstallFromURL()` 重新下载安装，保留用户配置和启用状态

## 插件市场 (Plugin Registry)

### 远程索引

插件市场复用 redc-template 仓库，通过 GitHub Actions 自动生成插件索引：

- 索引 URL：`https://redc.wgpsec.org/plugins/plugin-registry.json`
- 插件存放在 `redc-template/plugins/` 目录下
- GitHub Actions 构建时自动扫描插件目录，生成 `plugin-registry.json`，每个插件生成 ZIP 下载链接

### 前端版本比较

插件市场页面对每个插件显示三种状态：
- **未安装** → 显示「安装」按钮
- **已安装且最新** → 显示「已安装」标签
- **已安装但有新版** → 显示版本对比（`v1.0.0 → v1.0.1`）和「更新」按钮

版本比较使用 `utils/version.js` 中的 `compareVersions()` 函数。

## CLI 命令 (`cmd/plugin.go`)

```bash
redc plugin list                    # 列出已安装插件
redc plugin install <source>        # 安装（git URL 或本地路径）
redc plugin uninstall <name>        # 卸载
redc plugin enable <name>           # 启用
redc plugin disable <name>          # 禁用
redc plugin update <name>           # 更新（git pull 或 registry 重新下载）
redc plugin info <name>             # 查看详情
```

## 前端 (`PluginManager.svelte`)

侧边栏「系统」分组下的独立页面，包含两个视图：

### 已安装视图
- 统计卡片（总计 / 已启用 / 已禁用）
- 安装表单（Git URL 或本地路径输入）
- 已安装插件列表，每个插件卡片显示：
  - 名称、版本、分类标签
  - 启用/禁用开关（toggle）
  - 配置按钮（仅 config_schema 非空时显示）
  - 更新按钮、卸载按钮
- 配置弹窗：显示 config_schema 字段说明 + JSON 编辑器
- 卸载确认弹窗

### 插件市场视图
- 从 `redc.wgpsec.org` 拉取可用插件列表
- 每个插件显示名称、版本、描述、作者、标签
- 与本地已安装版本比较：未安装 / 已安装 / 可更新
- 手动安装表单（备用入口）

## 已迁移的内置模块

| 原内置模块 | 对应插件 | 功能 |
|-----------|---------|------|
| `gen_clash_config` | `redc-plugin-clash-config` | 从 terraform outputs 获取 IP，结合 tfvars 中的 port/password 生成 Clash 配置 |
| `upload_r2` | `redc-plugin-upload-r2` | 将 Clash 配置文件上传到 Cloudflare R2 |
| `chang_dns` | `redc-plugin-cloudflare-dns` | 修改 Cloudflare DNS 记录指向新 IP |

原 `mod/modules.go`（592 行）、`runModuleHook()` 方法、`case.json` 中的 `redc_module` 字段、约 40 个 `module_*` i18n 键均已删除。

## 安全注意事项

- 钩子脚本有任意代码执行权限，用户应在安装前审查插件代码
- `config.json` 可能包含敏感信息（API Key 等），存储在本地不上传
- 插件市场来源为 wgpsec 官方仓库，社区插件需经审核后合入
